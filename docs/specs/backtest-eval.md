# 规格：回测与评测框架（Backtest & Evaluation）

> 状态：草案（M3 定稿）。这是全项目最需严格验证的组件——评测错了，后面全错。

## 1. 目标

提供**可复现、抗过拟合、含真实成本**的评测，作为所有策略/信号改动的客观裁判（等价于软件项目的测试套件）。

- **Point-in-time 纪律**：只使用某时刻当时真实可得的数据快照。
- **防幸存者偏差**：包含已退市/下架标的。`TODO` 说明处理方式。
- 数据字典逐字段登记到 [../GLOSSARY.md](../GLOSSARY.md)。

## 3. 回测引擎

- **确定性/可复现**：固定随机种子、时间对齐、无未来函数（look-ahead）。
- **成本模型**：手续费、滑点、流动性/成交假设。参数：`TODO`。
- 记录运行元数据：数据版本、代码版本、参数、随机种子。

## 4. 过拟合防护（强制）

- Walk-forward 验证。
- Purged / embargoed 交叉验证（剔除信息泄漏样本）。
- 留出**最终验证集**，永不用于任何调参。
- 报告 Deflated Sharpe Ratio (DSR) 与 PBO。
- 记录"回测/调参次数"，用于多重检验惩罚。

## 5. 评测报告（一键生成）

- 指标：收益、最大回撤、Sharpe/Sortino、胜率、换手、成本、相对基准超额。
- 基准：SPY / BTC 买入持有。
- 输出可复现、可对比历史实验。

## 6. 完成定义（M3 退出标准）

- 同一输入两次运行结果一致。
- 基准报告可复现跑通。
- 过拟合防护在框架与流程中强制化。
