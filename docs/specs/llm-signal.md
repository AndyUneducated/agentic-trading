# 规格：LLM 信号层（LLM Signal Layer）

> 状态：草案（M2 定契约，M4 实现与评测）。核心：LLM 只产出结构化信号，绝不下单。

## 1. 职责边界

- 输入：非结构化信息（新闻、财报、公告、情绪等）。
- 输出：结构化信号/因子，交给决策层。
- 明确不做：不产生订单、不决定仓位、不触碰执行。

## 6. 已知失败模式与缓解（来自竞品实测，见 ../LANDSCAPE.md）

- **公开信息已定价**：新闻/财报的显性信息常已反映在价格中。缓解：聚焦时效性/解读增量，评测中强制对比价量基线。
- **LLM 保守偏差**：LLM 读过大量风险管理文献，倾向过度谨慎/看空。缓解：在 M4 评估偏差方向并在 prompt/校准层纠偏。
- **成本吃收益**：单票单次分析约 $0.30–0.50（GPT-5 级）。缓解：优先便宜后端（DeepSeek / 本地 Ollama，可降 80–90%）、低频、缓存。

## 7. 模型后端与多视角（可选）

- 记录并可切换后端；至少评测一个昂贵后端 vs 一个便宜后端的质量-成本权衡。
- 可选：借鉴 TradingAgents/ai-hedge-fund 的多视角（如多空/多人格）**信号提取**以降低单模型偏差——但**仍只产出信号，不下单**（不违反混合红线）。

## 8. 安全

- prompt 注入防护：外部文本输入需隔离与校验，不得让输入改变系统指令或触发越权动作。
