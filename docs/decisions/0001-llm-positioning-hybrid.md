# ADR-0001：LLM 定位采用混合架构（信号层，不下单）

- 状态：Accepted
- 日期：2026-07-14
- 决策者：项目所有者

## 背景（Context）

项目目标是可持续盈利，标的为美股/ETF/加密货币，先模拟盘验证再考虑小额实盘。需要确定 LLM 在交易闭环中的角色，因为它直接决定系统本质、验证难度与风险敞口。

关键约束：
- 交易后果不可逆（实盘亏损无法回滚）。
- LLM 输出具有不确定性、难复现，且读取外部非结构化信息时存在 prompt 注入风险。
- 目标偏向"求稳盈利"而非"前沿探索"。

## 备选方案（Options）

- **A 自主代理**：LLM 直接做交易决策并下单（observe→reason→decide→execute）。
  - 优点：更前沿、更"agentic"。
  - 缺点：验证难、可复现性差、风险敞口大，非机构主流。
- **B 纯信号层**：LLM 只输出因子，决策与执行完全交给传统量化，LLM 完全不碰交易动作。
  - 优点：最稳、最可控。
  - 缺点：未充分利用 LLM 的推理灵活性。
- **混合（Hybrid）**：LLM 做非结构化信息 → 结构化信号提取；硬规则/量化层负责决策与执行。
  - 优点：既利用 LLM 信息优势，又用确定性层守住风险与可复现性；更符合已验证的行业规范。
  - 缺点：需要维护信号层与决策层两套评测。

## 决策（Decision）

采用**混合架构**：LLM 作为信号层，负责从新闻/财报/公告/情绪中提取结构化信号/因子；决策与下单由确定性的规则/量化层完成。**运行时 LLM 绝不直接产生交易动作。**

## 后果（Consequences）

- 正面：风险可控、决策可复现、验证路径清晰；契合"求稳盈利"目标。
- 取舍：需要分别为"信号质量"（M4）与"策略盈利"（M3/M6）建立两层评测。
- 复核条件：若信号评测长期显示 LLM 无增量价值（Edge 假设被证伪），需回到 M2 重估；若未来要转向方案 A，须新开 ADR 取代本记录。
