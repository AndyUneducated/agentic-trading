# ADR-0003：回测-实盘一致（统一决策接口）

- 状态：Accepted
- 日期：2026-07-16
- 决策者：项目所有者

## 背景（Context）

竞品实测的最大教训之一：回测与实盘表现落差巨大。落差有两类来源——(1) 真实市场摩擦（滑点、成交、延迟），(2) **回测与实盘代码分叉**导致的行为差异。第二类是可以工程消除的。生产级系统（Nautilus Trader 同一套 actor/strategy 代码；FinRL-X 权重为中心的统一接口）都通过"同一套决策代码跑回测与实盘"来消除它，使 drift 只来自真实摩擦，从而可解释、可监控。见 [../LANDSCAPE.md](../LANDSCAPE.md)。

## 决策（Decision）

采用**统一决策接口**：定义单一的"信号 + 行情 → 目标仓位/订单"决策函数/接口，回测（M3）与模拟盘/实盘执行（M5+）**调用同一套决策代码**。执行环境差异（数据来源、下单通道）通过适配层隔离，决策逻辑本身零分叉。

## 后果（Consequences）

- M3 需在回测框架中就以该接口为核心；M5 的执行层复用它。
- live-vs-backtest drift 成为可信指标：偏离主要归因于真实摩擦，而非 bug。
- 约束：所有新策略/信号必须通过该接口接入，不允许为实盘另写一套决策逻辑。
- 与 ADR-0002 协同：若复用第三方引擎，需确认其支持回测-实盘一致的接口范式。
